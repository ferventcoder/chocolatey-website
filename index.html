The package manager for Windows

Builds on technologies you know

  • Chocolatey uses PowerShell automation scripts when working with installer architectures.
  • Because it is PowerShell, you can do almost anything.

Integrates with Windows install architectures

  • The Chocolatey PowerShell module provides great functions for working with all aspects of software.
  • Chocolatey can package or download software binaries, but it also works with just software embedded in the packages. Chocolatey works best when everything is embedded in the package.

Independent Deployment Artifacts

  • Chocolatey packages are not just fancy zip files, they understand versioning and dependencies, allowing you to encapsulate everything related to managing installation and configuration for a piece of software.
  • As an independent deployment artifact, Chocolatey packages are not tied to another technology, allowing developers to install packages with Chocolatey and System Administrators to deploy Chocolatey packages using their favorite Configuration Manager like SCCM, Puppet, or Chef.

Integrates with Configuration Managers (Plays well with others)

  • Chocolatey integrates with SCCM, Puppet, Chef, DSC and other configuration managers. This allows you to separate software installation concerns from those automation tools and just use the Chocolatey framework, which is fantastic at software management.
